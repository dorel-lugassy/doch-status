סקירה כללית

מערכת דוחות אקסל היא אפליקציית Python המבוססת על Streamlit ומיועדת לעיבוד, סינון וסיווג אוטומטי של הזמנות תקשורת (סיבים, נחושת ומוצרים נלווים כמו BIZNET ו-PHONE).

המערכת קוראת קובץ נתונים גולמי המופק ממערכות חיצוניות, מנקה אותו, וקובעת את סטטוס ההזמנה המדויק לכל לקוח. המטרה היא להפיק דוחות מפוצלים ומסודרים שמוכנים להמשך טיפול ולמנוע את הצורך במיזוג נתונים ידני.

מבנה הפרויקט

דוח סטטוס חדש/ ├── main.py ← ממשק המשתמש (Streamlit) ├── processors/ │ └── internet_morchav.py ← לוגיקת עיבוד הנתונים המרכזית └── utils/ └── excel_utils.py ← עזרים לטעינה ושמירת Excel
  • main.py: נקודת הכניסה לאפליקציה. מנהלת את מסך העלאת הקובץ, פיצול התוצאות לטבלאות תצוגה מקדימה, וכפתורי ההורדה למשתמש.
  • internet_morchav.py: ליבת המערכת העסקית. הקובץ מכיל את פונקציית run() אשר מריצה את אלגוריתם סינון השורות, המיפוי של מוצרים נלווים (מגיליון "כל השאר") וקביעת הסטטוסים.
  • excel_utils.py: שכבת התשתית המטפלת בקריאה וכתיבה של DataFrames לאקסל ובניקוי רווחים מיותרים בטקסטים.

לוגיקה עסקית ליבה

המערכת מחברת יחד את גיליונות סיבים ו-נחושת לטבלה ראשית אחת.

  • שורה אחרונה קובעת: כיוון שמספר הזמנה עשוי להופיע מספר פעמים עם עדכונים שונים, הקוד מתקבץ לפי מספר הזמנה ולוקח את השורה האחרונה ביותר המופיעה בקובץ.
  • קביעת סטטוס: הסטטוס נגזר מעמודת סטטוס שירות:
    • סגור: תאריך היעד יילקח מעמודת תאריך סגירת הזמנה.
    • פתוח: תאריך היעד יילקח מעמודת תאום לקוח התחלה.
    • מבוטל: ההזמנה מבוטלת.
    • ערכים אחרים נחשבים חריגים ומועברים לגיליון ייעודי (מניעת קריסה).

במקביל להזמנה הראשית, לקוחות מזמינים מוצרים נלווים. אלה נמצאים בגיליון השלישי:

  • BIZNET (מוצר משלים): מחלצים שורות בהן שירות = BIZNET. הן מוזרקות בחזרה אל טבלת האינטרנט הראשית. התאריך במקרה זה נשאב מעמודת פרטי תאום לקוח במשימה אחרונה בהוראת עבודה.
  • PHONE (קו טלפון): שורות בהן שירות = PHONE. כדי להיחשב לקו טלפון תקין לדוח, חובה שיהיה ערך בעמודת חבילת דקות. מוצרים אלו יוצאים לקובץ אקסל נפרד לחלוטין.

כיוון שהשירותים המשלימים (BIZNET ו-PHONE) שייכים לאותה הזמנה של תשתית האינטרנט, המערכת מיישמת לוגיקה של הורשת סטטוס:

לפני שהמערכת כותבת את הפלט הסופי של מוצרי BIZNET ו-PHONE, היא בודקת:

  • האם מספר ההזמנה שלהם מופיע במאגר הראשי (נחושת/סיבים)?
  • אם כן – הם ידרסו את הסטטוס העצמאי שלהם ויקבלו את הסטטוס של ההזמנה הראשית.
  • לדוגמה: אם BIZNET הגיע כ"מבוטל" אך הזמנת הנחושת הופעלה וסומנה כ"סגור", השורה של ה-BIZNET תירשום גם היא "סגור", שכן ההזמנה עצמה (והתשתית בפרט) סגורה.

קלט ופלט

עמודות הקלט הנדרשות

קובץ ה-Excel המקורי כולל בתוכו שורות כותרת כפולות, לכן הקוד מוגדר לחפש את הכותרות האמיתיות בשורה 2. עמודות שחובה שיופיעו:

  • גיליונות תשתית: מספר הזמנה, מספר הכר, סטטוס שירות, תאום לקוח התחלה, תאריך סגירת הזמנה.
  • גיליון כל השאר: שירות, סטטוס שירות, פרטי תאום לקוח במשימה אחרונה בהוראת עבודה, חבילת דקות.

קובצי הפלט

האפליקציה מחלקת את התוצרים ל-3 קבצים להורדה:

  • סטטוס אינטרנט מחודש להרצה - עם תאריך מתואם (מכיל רק הזמנות שלקוח תואם איתן).
  • סטטוס אינטרנט מחודש להרצה - ללא תאריך מתואם (מכיל שורות ריקות מתאריך עבור טיפול של הנציגים).
  • סטטוס קו טלפון (מכיל לקוחות PHONE בלבד).
